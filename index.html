OpenSCAD 101: The Architect's CAD

OpenSCAD isn't your typical click-and-drag CAD program. It's a programmable modeling tool that thinks like an architect's blueprint—parametric, precise, and infinitely modifiable.

Why OpenSCAD? Unlike traditional CAD, every dimension is a variable. Change one parameter, and your entire design updates intelligently. Perfect for creating families of related objects or fine-tuning fits.

Core Philosophy: Code as Blueprint

Think of OpenSCAD code as architectural drawings—clear, documented, and parametric:

// Device dimensions (like architectural specs)
device_w = 148;   // left‑right width
device_h = 86;    // vertical height
device_d = 73;    // front‑back depth
clear    = 1.0;  // fit clearance (adjust if tight)

// Create a parametric box
cube([device_w + 2*clear, device_h + 2*clear, device_d]);

Basic Building Blocks

OpenSCAD provides primitive shapes that you combine using boolean operations:

// Primitives
cube([10, 20, 5]);        // rectangular box
cylinder(h=10, d=8);      // cylinder (height, diameter)
sphere(r=5);               // sphere (radius)

// Boolean operations
union() {              // combine shapes
    cube([10, 10, 5]);
    cylinder(h=5, d=6);
}

difference() {         // subtract second from first
    cube([10, 10, 5]);
    cylinder(h=6, d=3); // creates hole
}

Key Concept: OpenSCAD uses Constructive Solid Geometry (CSG). You build complex shapes by combining, subtracting, and intersecting simple ones—just like architectural massing studies.

Fundamental Concepts

Parametric Design Thinking

Always start with parameters at the top of your file. This makes your designs adaptable and self-documenting:

// Junction box geometry (real-world constraints)
box_slot_center = 89;      // ~3.5" on-center screw spacing
box_slot_len    = 18;      // slot length for tolerance
socket_clear_d  = 52;      // E26 ceramic socket clearance

// Material considerations
wall_thickness = 3.2;      // minimum for structural integrity
print_tolerance = 0.2;     // 3D printer fit allowance

Transformations: Moving Things Around

OpenSCAD uses transformations to position and orient objects:

// Position, rotate, scale
translate([10, 20, 5])    // move to position
    rotate([0, 0, 45])       // rotate 45° around Z
        cube([5, 5, 10]);

// Center objects for easier positioning
cube([10, 10, 5], center=true);  // centered on origin

Modules: Reusable Components

Modules are like functions—they encapsulate geometry and can be parameterized:

// Define a reusable mounting tab
module mounting_tab(width=14, height=10, thickness=3.2) {
    difference() {
        cube([width, thickness, height], center=true);
        // Screw hole
        cylinder(h=thickness+0.6, d=3.2, center=true);
    }
}

// Use it multiple times
for (side=[-1, 1]) {
    translate([side * 50, 0, 0])
        mounting_tab();
}

Echo Show Mount: Parametric Device Cradle

This project showcases how to design around real-world constraints—the device dimensions drive everything:

// Start with the thing you're designing FOR
device_w = 148;   // Echo Show 5 width (measured)
device_h = 86;    // height
device_d = 73;    // depth

// Design decisions become parameters
clear = 1.0;       // how much clearance? (user adjustable!)
lip_h = 5;        // front lip to prevent sliding
cheek_t = 3.6;     // side wall thickness

// The cradle dimensions are calculated, not hardcoded
cradle_w = device_w + 2*clear + 2*cheek_t;
cradle_h = device_h + 2*clear + lip_h;

Design Philosophy: Notice how dimensions cascade. Change the device size or clearance, and the entire cradle updates. This is parametric design thinking—relationships, not just measurements.

Advanced Technique: Detent Mechanism

The hinge uses micro-teeth for positioning feedback—a complex feature made simple with parameterization:

module detent_ring(r=7.5, teeth=12, depth=0.6) {
    difference() {
        cylinder(h=1.0, r=r, center=true);
        
        // Carve triangular teeth around the circumference
        for (i=[0:teeth-1]) {
            angle = 360/teeth * i;
            rotate([0, 0, angle])
                translate([r - depth/2, 0, 0])
                    // Triangular tooth profile
                    polygon(points=[[0,0], [depth, depth/2], [depth, -depth/2]]);
        }
    }
}

Advanced Techniques

Print-Conscious Design

Your OpenSCAD designs need to work with 3D printing constraints:

// Design for 3D printing success
nozzle_size = 0.6;           // drives minimum feature size
layer_height = 0.28;         // affects vertical resolution
min_wall = nozzle_size * 4;   // 4 perimeters for strength

// Overhangs: design to avoid supports
max_overhang_angle = 45;     // degrees from vertical

// Tolerance for moving parts
print_clearance = 0.2;       // gap for sliding fits
thread_clearance = 0.1;      // tighter for threaded parts

Minkowski and Rounded Features

Create professional-looking rounded edges without complex math:

// Rounded box using minkowski
module rounded_box(x, y, z, r=2) {
    minkowski() {
        cube([x-2*r, y-2*r, z-r]);
        sphere(r=r);  // "inflates" all edges by radius r
    }
}

// Usage: creates a 20x10x5 box with 2mm rounded edges
rounded_box(20, 10, 5, r=2);

Conditional Design Elements

Make features optional with conditional logic:

// Optional keyhole slots (set to 0 to disable)
keyhole_enabled = true;
keyhole_w = keyhole_enabled ? 8 : 0;

if (keyhole_w > 0) {
    // Only generate keyhole geometry if enabled
    for (x=[-20, 20]) {
        translate([x, 0, 0])
            keyhole_slot(keyhole_w);
    }
}

Professional Workflow

File Organization

Structure your OpenSCAD projects like architectural drawings:

// Header: Project info and licensing
// Echo Show 5 Mount — Parametric OpenSCAD
// Author: Your Name
// License: CC-BY 4.0

// Parameters section (like a spec sheet)
// --- DEVICE DIMENSIONS ---
device_w = 148;

// --- PRINT SETTINGS ---
layer_height = 0.28;

// Utility modules (reusable functions)
module rounded_box(...) { ... }

// Main components
module main_body() { ... }
module mounting_plate() { ... }

// Layout/preview logic
if (ASSEMBLY_VIEW) assembly(); else parts_for_printing();

Testing and Iteration

Design verification strategies:

Test Print Strategy: Print small test sections first—screw holes, fit tolerances, hinge mechanisms. Don't print the full part until you've validated critical dimensions.

Parameter Tuning

Make your designs user-adjustable:

// Tuning section (document what users might need to adjust)
// --- TUNING PARAMETERS ---
fit_adjustment = 0.0;    // add if parts are tight (+0.5)
hinge_tension = 1.0;     // 0.8=loose, 1.2=tight
support_free = true;   // false adds support structures

// Apply adjustments to base measurements
actual_clearance = base_clearance + fit_adjustment;

Safety and Documentation

For electrical or mechanical projects, embed safety requirements:

// SAFETY: LED bulbs only - max 15W
// Mount to code-compliant junction box
// Maintain minimum clearances for heat dissipation

max_bulb_wattage = 15;        // Watts (LED only)
min_ventilation = 20;        // mm² vent area per Watt
heat_clearance = 10;         // mm from any heat source

Professional Responsibility: When designing electrical enclosures or mechanical parts, research applicable codes and safety standards. Your parametric design should enforce safe practices, not just aesthetic choices.
